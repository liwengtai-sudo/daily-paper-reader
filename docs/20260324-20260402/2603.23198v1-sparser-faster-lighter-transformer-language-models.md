## 摘要
扩展自回归大语言模型（LLMs）推动了前所未有的进步，但也带来了巨大的计算成本。在这项工作中，我们通过利用 LLM 前馈层中的非结构化稀疏性来解决这些成本问题，前馈层是模型中占据绝大部分参数和执行浮点运算量（FLOPs）的组件。为了实现这一目标，我们引入了一种新的稀疏打包格式和一组 CUDA 内核，旨在与现代 GPU 的优化执行流水线无缝集成，从而在 LLM 推理和训练期间实现高效的稀疏计算。为了证实我们的收益，我们对 LLM 稀疏性进行了定量研究，证明简单的 L1 正则化可以诱导超过 99% 的稀疏性，且对下游性能的影响微乎其微。通过与我们的内核结合，我们展示了这些稀疏水平可以转化为显著的吞吐量、能效和内存使用优势，且这些优势随模型规模的扩大而增加。我们将以开源许可证发布所有代码和内核，以促进采用并加速相关研究，旨在将稀疏性确立为提高现代基础模型效率和可扩展性的一个实用维度。

## Abstract
Scaling autoregressive large language models (LLMs) has driven unprecedented progress but comes with vast computational costs. In this work, we tackle these costs by leveraging unstructured sparsity within an LLM's feedforward layers, the components accounting for most of the model parameters and execution FLOPs. To achieve this, we introduce a new sparse packing format and a set of CUDA kernels designed to seamlessly integrate with the optimized execution pipelines of modern GPUs, enabling efficient sparse computation during LLM inference and training. To substantiate our gains, we provide a quantitative study of LLM sparsity, demonstrating that simple L1 regularization can induce over 99% sparsity with negligible impact on downstream performance. When paired with our kernels, we show that these sparsity levels translate into substantial throughput, energy efficiency, and memory usage benefits that increase with model scale. We will release all code and kernels under an open-source license to promote adoption and accelerate research toward establishing sparsity as a practical axis for improving the efficiency and scalability of modern foundation models.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：尽管大语言模型（LLM）的扩展带来了性能飞跃，但其计算成本、能耗和内存需求也随之激增。前馈网络（FFN）层占据了模型约 2/3 的参数和超过 80% 的计算量（FLOPs）。
*   **研究背景**：虽然研究表明 FFN 层存在天然的激活稀疏性，但由于现代 GPU 架构（如 NVIDIA Hopper）是为密集矩阵运算高度优化的，非结构化稀疏操作在实际运行中往往比密集运算更慢。
*   **研究动机**：开发一种能够与现代 GPU 硬件流水线无缝集成的稀疏计算方案，打破“稀疏性在理论上高效但在实践中低效”的僵局，使非结构化稀疏性成为提升模型效率的实用手段。

### 2. 论文提出的方法论
核心思想是通过新型数据格式和定制 CUDA 内核，在不改变主流训练配方的前提下，实现端到端的稀疏加速。
*   **TwELL (Tile-wise ELLPACK) 格式**：
    *   **创新点**：传统的 ELL 格式需要全局同步来对齐非零元素，而 TwELL 将列划分为 1D Tile（如 256 维度）。
    *   **优势**：它可以在矩阵乘法内核的收尾阶段（Epilogue）直接生成，无需额外的内核启动或全局内存同步，极大地降低了格式转换开销。
*   **推理内核融合**：
    *   将 Gated FFN 的三个投影（Gate, Up, Down）融合。利用 Gate 产生的 TwELL 模式，仅对非零激活对应的权重进行点积运算，大幅减少 DRAM 访问。
*   **训练内核与 Hybrid 格式**：
    *   **Hybrid 格式**：针对训练中稀疏模式极度不均（某些 Token 激活多，某些少）的问题，将行动态划分为“紧凑 ELL 存储”和“密集备份存储”。
    *   **反向传播优化**：设计了支持 Hybrid 格式转置和梯度注入的内核，使反向传播也能从稀疏性中获益，显著降低中间激活值的显存占用。
*   **稀疏诱导方案**：采用简单的 ReLU 激活函数配合轻量级的 L1 正则化损失，强制模型在训练过程中产生稀疏激活。

### 6. 主要结论与发现
*   **极高稀疏度**：通过轻微的 L1 正则化，可以在几乎不损失下游任务性能的情况下，实现 **>99%** 的激活稀疏度。
*   **显著提速**：在 2B 参数规模下，稀疏内核使推理吞吐量提升 **20.5%**，训练速度提升 **21.9%**。
*   **资源节省**：训练时的峰值显存占用减少了约 **28%**，且能耗显著降低。
*   **扩展性规律**：模型规模越大，稀疏性带来的收益越明显，这符合当前大模型发展的长远趋势。
*   **稀疏模式发现**：模型倾向于在处理高信息量 Token（如特定名词、序列起始位置）时分配更多激活，而在处理预测性强的 Token（如常见连接词）时保持极高稀疏。

### 7. 优点
*   **工程落地性极强**：不同于许多仅停留在理论 FLOPs 减少的研究，本文提供了实测的端到端 GPU 加速数据，并开源了 CUDA 内核。
*   **全生命周期优化**：同时覆盖了训练和推理阶段，特别是 Hybrid 格式解决了稀疏训练中负载不均衡的痛点。
*   **硬件感知设计**：TwELL 格式的设计充分利用了 Hopper 架构的 TMA 和异步拷贝特性，展现了深厚的底层优化功底。

### 8. 不足与局限
*   **架构依赖**：目前的内核高度针对 NVIDIA Hopper 架构优化，在其他架构（如 AMD 或旧版 NVIDIA GPU）上的移植效果尚待验证。
*   **死神经元风险**：高强度 L1 正则化会导致部分神经元永久失活，虽然文中提出了重初始化策略，但这增加了训练的复杂性。
*   **模型规模上限**：实验最高仅做到 2B 参数，虽然趋势向好，但在百亿或千亿级参数模型上的表现（如是否存在训练稳定性问题）仍需进一步验证。
*   **激活函数限制**：必须使用 ReLU 族激活函数，对于习惯使用 SiLU/SwiGLU 的现有预训练模型，需要进行微调或持续预训练才能适配。
